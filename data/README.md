# AegisShield Dataset Guide

This folder contains raw datasets, processed outputs, and sample assets used by the misinformation detection pipeline.

## Folder Layout

- `datasets/`: Raw source datasets used for training and preprocessing
- `processed/`: Cleaned and merged CSV files generated by preprocessing scripts
- `samples/`: Sample files (for example, OCR test images)

## Dataset Sources

1. Google Drive Folder (project data bundle)
	- Link: https://drive.google.com/drive/folders/1U0ODCW5llnrm3wKSuDYF2uR4SPdYwFMs?usp=sharing
	- Usage: Consolidated/shared project assets and dataset files used in development

	Note: The Google Drive folder above contains the complete copies of both the `datasets/` and `processed/` folders (raw source files and processed outputs). Use the same link to access either folder.

2. LIAR Dataset (Kaggle)
	- Link: https://www.kaggle.com/datasets/doanquanvietnamca/liar-dataset
	- Local files expected in `datasets/`: `train.tsv`, `valid.tsv`, `test.tsv`
	- Notes: Used for statement-level misinformation labeling

3. Fake and Real News Dataset (Kaggle)
	- Link: https://www.kaggle.com/datasets/clmentbisaillon/fake-and-real-news-dataset
	- Local files expected in `datasets/`: `Fake.csv`, `True.csv`
	- Notes: Used for article-level fake/real news classification

## How Processed Files Are Generated

Run the backend scripts in this order:

1. `backend/scripts/process_news_dataset.py`
2. `backend/scripts/process_liar_dataset.py`
3. `backend/scripts/combine_datasets.py`

Outputs are saved to:

- `processed/news_processed.csv`
- `processed/liar_processed.csv`
- `processed/combined_dataset.csv`
